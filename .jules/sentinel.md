## 2024-05-24 - Client-side Mailchimp API Key Exposure & Reverse Tabnabbing
**Vulnerability:** Found a client-side `fetch` call to a Mailchimp API in the `Footer.tsx` newsletter signup form that included an API key hardcoded within the Authorization header (`Authorization: "apikey YOUR_API_KEY"`). Additionally, an external link to `https://gohrvst.com` had `target="_blank"` without `rel="noopener noreferrer"`.
**Learning:** Third-party integrations requiring API keys or secrets must never be executed directly from client-side React code. They should be proxied through a backend API endpoint where the secret can be securely stored. Leaving off `noopener noreferrer` allows the new tab to hijack the original tab's execution context.
**Prevention:** Always implement a backend proxy for API calls requiring sensitive secrets. Use `.env` variables strictly for public keys (`VITE_PUBLIC_...`), never for private keys. Always pair `target="_blank"` with `rel="noopener noreferrer"`.
