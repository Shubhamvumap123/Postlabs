# Palette's Journal - Critical Learnings

## 2026-02-25 - Context-Aware Empty States
**Learning:** Using generic empty states (e.g., "No tasks") across different tabs (Scheduled vs Completed) creates confusion. Context-specific messaging (e.g., "No completed tasks yet") clarifies system state.
**Action:** When implementing tabbed interfaces, always define a map of empty states for each tab key to ensure relevance.
