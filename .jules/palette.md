## 2024-05-18 - Missing Accessibility Attributes on Custom Dialog Components
**Learning:** Custom dialog components missing `role="dialog"`, `aria-modal="true"`, and appropriate `aria-labelledby`/`aria-describedby` attributes fail severely on screen readers and cannot be identified as modal dialogs. Additionally, relying on `framer-motion` for entry/exit animations without implementing robust native event listeners (like global `keydown` listeners on the `document`) prevents keyboard-only users from closing the dialog using the `Escape` key.
**Action:** When implementing custom modal components instead of using verified primitives (like Radix UI), ensure all mandatory ARIA roles and keyboard event listeners (`Escape` key specifically attached to the `document` with a discrete `useEffect`) are implemented. Furthermore, ensure interactive controls (like the close 'X' button) always have `focus-visible` states and programmatic `aria-label` text.
