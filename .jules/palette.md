## 2024-05-18 - Keyboard Accessibility for Hover-Only Actions
**Learning:** Elements relying on `group-hover` for visibility (like inline task actions) become invisible traps for keyboard users unless they also receive `focus-within` styles. Using `group-hover:opacity-100` alongside `focus-within:opacity-100` ensures actions become visible during Tab navigation. Furthermore, low-contrast text on dark backgrounds (`bg-zinc-900`) needs lighter shades (e.g., `text-zinc-400` instead of `text-zinc-500`) to meet WCAG AA standards.
**Action:** When adding hover-only UI elements, always include a `focus-within` counterpart for the parent container and `focus-visible:ring` for the interactive child elements to guarantee keyboard navigability. Always verify contrast ratios for text on dark mode panels.
