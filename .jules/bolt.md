## 2024-05-18 - Vite Chunk Size Warning Resolution
**Learning:** Vite bundles the entire application into a single large chunk by default, which can cause slow initial load times and trigger chunk size warnings (>500kB) during production builds. The `postlabs` project was suffering from this issue because all top-level route pages were imported synchronously in `src/App.tsx`.
**Action:** Implemented route-level code splitting using `React.lazy` and `Suspense`. By dynamically importing the page components (`lazy(() => import("./pages/Index"))`) and wrapping the React Router `<Routes>` block in a `<Suspense>` component with a `LoadingFallback`, Vite now splits each route into its own smaller chunk. This significantly reduces the main bundle size, improving Time to Interactive (TTI) and silencing the Vite chunk size warning.
